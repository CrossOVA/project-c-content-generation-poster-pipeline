# Генерация и публикация контента Project C

Дочерний сценарий n8n для генерации текстового контента Проекта C, создания изображения, публикации в X и Telegram и сохранения результатов в таблицу.

## Что делает сценарий

Сценарий запускается из основного контент-оркестратора, получает исходный инфоповод, уже подготовленный контекст и режим публикации для Проекта C.

В зависимости от режима он генерирует либо образовательный пост, либо новостной пост. После этого сценарий разбирает JSON-ответ модели, готовит текст для X и Telegram, создаёт изображение, публикует материалы в каналы, сохраняет ссылки и тексты в Гугл-Таблицы и отправляет уведомление об успешной публикации администратору в Telegram.

Если на одном из этапов возникает ошибка, сценарий отправляет её в Telegram вместе с кратким AI-разбором причины.

## Что реализовано

- задержка публикации относительно других проектов;
- выбор режима публикации: EDU или NEWS;
- чтение RAG-списка разрешённых упоминаний из Google Sheets;
- подготовка списка валидных X-аккаунтов для AI-модели;
- генерация текста для X;
- генерация текста для Telegram;
- генерация заголовка и промпта для изображения;
- строгий JSON-формат ответа модели;
- проверка успешного парсинга JSON;
- удаление повторных тикеров в Telegram-тексте посредством code-узла;
- генерация изображения для поста;
- загрузка изображения во внешний сервис публикации;
- публикация поста с изображением в X через внешний сервис;
- проверка статуса публикации через внешний сервис;
- публикация поста с изображением в Telegram;
- сохранение X-текста, X-ссылки, Telegram-текста и Telegram-ссылки в Google Sheets;
- Telegram-уведомление об успешной публикации;
- отдельные ветки обработки ошибок для генерации изображения, загрузки изображения, публикации в X, публикации в Telegram и записи в таблицу;
- ИИ-анализ ошибок с отправкой результата в Telegram.

## Мой вклад

Разработал и собрал дочерний сценарий для Проекта C, настроил его запуск из основного контент-оркестратора и добавил разделение на два режима публикации: образовательный и новостной.

Написал JavaScript-логику для подготовки списка валидных упоминаний, разбора JSON-ответа модели, проверки результата генерации и очистки Telegram-текста от повторных тикеров.

Также настроил генерацию текстового контента через ИИ-модель, генерацию изображения, публикацию в X и Telegram, запись результатов в Гугл-Таблицы и ветки обработки ошибок. Отдельно добавил Telegram-уведомления и ИИ-разбор ошибок, чтобы сбои не терялись и были понятны без ручного разбора рабочего потока.
